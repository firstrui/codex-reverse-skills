## 先分类型，不要一把梭

从 `1997.pro` 语料里可以反复看到一个原则：

- 先分清是 `token lane`
- 还是 `sign lane`
- 还是 `fingerprint lane`
- 还是 `challenge / captcha lane`

常见误判是：把服务端发回的短效令牌，当成客户端纯算签名去抠；或者把真正的问题是 TLS / cookie / challenge 状态缺口，误判成 JS 算法没还原。

## 指纹面不要只看对象名

高频指纹面包括：

- DOM / BOM
- `canvas` / `webgl` / `plugins`
- 存储能力与异步接口，如 `navigator.storage.estimate()`
- 时间源、随机源、运行时格式化特征
- TLS / JA3 / JA4 / ClientHello
- 鼠标 / 行为轨迹

经验要点：

- 不要只罗列“检测了哪些字段”，要看最终是谁在消费这些字段。
- 很多样本只是把采集数组再做加密和拼接，大 `switch` 本身不一定是核心。
- 浏览器能出值而本地 403，不一定是 JS 缺对象，也可能是 TLS 指纹或上游 cookie 状态差异。

## 厂商族谱的高价值线索

### Akamai

优先怀疑：

- 动态字符串解密
- 最终加密栈附近的真实入口
- 版本轮换但结构相近的环境载荷

可复用经验：

- 先断到最终加密或最终打包附近，再向上一个栈取原始入参。
- 动态字符串解密函数要等“全部初始化完成后”再调；中途断点常常拿不到完整映射。
- 如果脚本十几分钟切一份，先冻结一份样本，再做反混淆和字符串回填。

### Kasada / 5 秒盾 / worker 通信型风控

优先怀疑：

- `iframe` / `worker`
- `postMessage` / `MessageChannel` / `BroadcastChannel`
- 跨上下文消息与功能性检测

可复用经验：

- 先证明“消息通道有没有参与状态流转”，再谈补 `window`。
- 如果浏览器自动化工具在 `run_js` 或通道功能检测上暴露，先做最小定向修补。

### PX / Reese84 / Incapsula

优先怀疑：

- wasm + JS 双层链路
- 按压 / 交互行为字段与风险 cookie 耦合
- 资源拉取与 challenge / resource 分支

可复用经验：

- 不要把 `_px3` 或 `reese84` 当成单一纯算值，先看上游资源请求和 cookie hop。
- 如果字段由 wasm 参与生成，先记录 wasm 下载、实例化和导入对象。

### 同盾 BlackBox

优先怀疑：

- 环境数组先采样、后统一加密装配
- 多套检测数组顺序打乱，但核心检测点复用
- `storage.estimate()` 这类异步接口

可复用经验：

- 先找“最终数组采样点”，不要先深挖后面的拼接循环。
- 时间检测、格式化检测、异步能力检测经常比表面对像更关键。

### 字节 `a_bogus`

优先怀疑：

- 版本号与 salt 漂移
- 内核未彻底重写，只是局部魔改
- 旧版 purecalc 可以作为差分基线

可复用经验：

- 先拿旧版已通样本与新版对比，优先找 `salt`、短数组、版本号、RC4 类内核变化。
- 插桩日志规模可控时，不一定要完整反编译，先修出可用新版本。

## TLS / JA4 是单独一层

从 `TLS/JA4` 与 `TLS&&补环境理解` 两篇里可以抽出一条硬规则：

- JS 参数对了，不代表网络栈也对了。

遇到以下现象时优先怀疑 TLS 层：

- 浏览器通、Python/Node 客户端稳定风控
- 参数回放一致，但响应分支仍然不同
- 站点明显依赖 CDN / WAF 的握手特征

优先动作：

- 抓 `ClientHello`
- 记录浏览器与客户端的套件、扩展、签名算法差异
- 把 TLS 差异和 JS 差异分开记录

## 行为风控不是只补一条轨迹

行为类文章给出的启发是：

- 轨迹不仅是几何形状，还包括时间、抖动、停顿、采样密度、加速度
- 先看服务端吃哪些字段，再决定是规则模拟、统计拟合还是生成式方案

在实战里：

- 优先做确定性回放与字段拆分
- 只有当规则型手段已明显不够时，才引入更重的生成式方法

## 起手证据点

优先产出这三类证据：

- 真实请求 / challenge / cookie hop 链
- 被消费的指纹载体，而不是“所有可见指纹面”
- 浏览器与本地在 TLS、cookie、storage、时间源上的第一个真实分叉

## 推荐 handoff

- 阶段不明：`$web-js-reverse-master-flow`
- 请求链未坐实：`$jsr-locate`
- 已知浏览器能过、本地不过：`$jsr-runtime`
- 已知是环境拟合：`$env-patch`
- 已知有 challenge / slider 双链：`$captcha-parameter-chain-bisection`
